1. **层序遍历的辅助队列大小应该如何设置** -- gzh

   只有在**完全二叉树中**，最后一行的节点数才可能最多。对于有 **`n`** 个节点的完全二叉树：

   - 如果树是满二叉树，最后一行的节点数 = 总节点数的一半（向上取整）

   - 具体公式：当树接近满二叉树时，最后一行节点数 ≈ **`n/2`**（向上取整）

     ```c
     证明:
     以n=2000举例
        							n = n0 + n1 + n2
     							n = n1 + 2n2 + 1
     联立得：n0 = n2 + 1
     假设n1最少，则n0+n2为2000或者近似于2000，所以最多只有n0 = 1000，n2有999个，还有一个n1结点
     ```

2. **waitpid函数** -- cjs

   ```c
   waitpid函数的作用是等待一个已经退出的子进程，并进行清理工作。
   
   waitpid 等待一个指定退出的子进程，并返回该子进程的PID
   waitpid 是一个阻塞函数
   pid_t waitpid(
       pid_t pid,      // 指定等待的PID的子进程
       int *wstatus,   // 存储进程的退出状态
       int options     // 修改 waitpid 的行为的选项, 默认0
   );
   PID数值	效果
   < -1	等待进程PID和pid绝对值的进程
   == -1	等待任一个子进程, 等价于wait
   == 0 	等待同一进程组的任意子进程
   waitpid是阻塞函数，如果给waitpid 的options参数设置一个名为WNOHANG的宏，则系统调用会变成非阻塞模式。
   ```

3. **fork以及exec函数** -- xkz

   在一般情况下，我们通常用fork配合exec函数使用，并在父进程中wait等待，因此统一讨论

   1.fork()  系统调用，通过复制一个现有的进程来创建一个全新的进程。调用的原进程称为父进程，新进程称为子进程，因此存在两次系统调用的从内核返回，也即返回原进程和子进程

   在创建的过程中，父子进程共享代码段，但无法共享数据段，堆栈，如果在创建时就进行复制，不仅会产生高额的读写性能开销，而且，fork的子进程通常要通过exec调用启动调用程序，这是会覆盖掉与原来的数据提供给新程序。因此如果创建时复制，性能代价大。

   ​	因此使用了写实拷贝的技术，创建后临时指向同一个物理内存页，对该物理内存页实行写保护，一旦由一方因为写保护触发了缺页异常，内核为该页创建一个新的物理页面，此时才由真正的独立页面，并且撤销写保护。

   ​	在这个过程中，哪个进程触发的写保护就为谁分配新物理页，另一个进程不变，但由于写保护的关闭，此物理页的内容以及独立。

   这个过程缺页异常的过程

   1.缺页异常最终调用do_pape_fault

   2.进而调用handle_mm_fault

   3.再调用handle_pte_fault

   4.调用do_wp_pape  该函数会创建副本

   ​		同时也涉及到，父子进程运行先后的问题，这个决断需要结合实际的情况处理

   父进程先运行的好处在于，fork前后，父进程在CPU处于活跃状态，CPU的硬件内存管理单元TLB中，目前缓存的都是父进程的页表项，因此父进程继续执行，处理器性能更高

   子进程先运行的好处在于，如果父进程运行，大概率要触发写保护，进而进行复制页，而子进程创建后通常很快就调用exec了，如果父进程先运行，进行了大量复制，而子进程运行后，这些复制页大可能会被完全覆盖掉，如果先调用子函数，能节省这部分潜在的性能开销
