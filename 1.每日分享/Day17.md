3. **管道|的应用**

   cmd1 | cmd 2 就表示将cmd1指令执行的输出结果，作为cmd2指令的输入。

   ​		管道一般配合xargs进行使用 ：格式为 cmd1 | xargs cmd2 ;xargs的效果就是输入一行执行一次命令

   ​		含义为：将cmd1指令输出的每一行，作为参数传递给cmd2指令进程。

   ​		其最主要的一个场景：在linux中有很多类型别名或者结构体定义，其在预处理阶段会进行一个替换的操作，此时可以通过此操作来进行查询。

   ​		例子:查找printf在 /usr/include目录下的哪一个.h文件中

   ​				find /usr/include/ -name "*.h" |xargs grep -nE "int printf"

    区别:

   ```
   find | grep
   ```

   = grep 搜索 **文件名字符串**

   ```
   find | xargs grep
   ```

   = grep 搜索 **这些文件的内容**

4. **正则表达式**

   | 序号 | 符号 / 写法 | 作用说明                                |
   | ---- | ----------- | --------------------------------------- |
   | 1    | ""          | 内容包含匹配                            |
   | 2    | .           | 任意单个字符匹配                        |
   | 3    | *           | 前一个字符任意次数匹配                  |
   | 4    | .*          | 任意内容、任意数量字符匹配              |
   | 5    | \           | 转义，用于特殊字符直接匹配              |
   | 6    | ()          | 分组，将括号内内容作为整体匹配          |
   | 7    | []          | 匹配括号中任意单个字符                  |
   | 8    | [^...]      | 括号内用 ^ 表示取反，匹配排除其中的内容 |
   | 9    | {m,n}       | 前面内容重复 m 到 n 次                  |
   | 10   | {m}         | 前面内容只重复 m 次                     |
   | 11   | {m,}        | 前面内容重复 m 次到正无穷               |
   | 12   | ^           | 匹配行头                                |
   | 13   | $           | 匹配行尾                                |
   | 14   | <  >        | 单个单词的边界匹配                      |
   | 15   | ?           | 前一个字符匹配 0 次或 1 次              |
   | 16   | +           | 前一部分内容至少出现 1 次               |
