1. **vim与xshell的配置文件修改**

   ​		在xshell与vim的实际使用中，会发现默认设置极其难用，但是有许多扩展命令可以给我们使用，但是每次命令生效之后只在此次连接中有效，总不能一直保持电脑为开机状态，所以可以通过修改配置文件来使效果永久生效，这是因为每次xshell与vim运行前都会先执行它们自己的配置文件，并且经过我的使用测试，每个用户之前的配置文件都是独立的，这两个工具的配置文件都在每个用户的家目录之下。

   ```c
   // 例如：我的用户名gzh的xshell与vim的配置文件的绝对路径就分别是:
   /home/gzh/.bashrc
   /home/gzh/.vimrc
   ```

   操作流程为：

   1. 先用vim打开xshell配置文件

      ```c
      vim ~/.bashrc
      ```

   2. 通过vim命令G将光标移动到最后一行，在下面插入需要添加的命令，例如我添加的：

      ```c
      # 设置用户名与当前目录颜色，并隐藏主机名——用户名蓝色，当前目录绿色
      export PS1="\[\033[34m\]\u\[\033[0m\]:\[\033[32m\]\W\[\033[0m\]\$ "
      
      # 设置 man 手册默认语言为中文——但是多数函数还是没有中文版介绍
      export LC_MESSAGES=zh_CN.UTF-8
      export MANLANG=zh_CN
      
      # 自定义别名——输入llh即可查看当前目录所有文件的所有信息
      alias llh='ls -alF -i -h'
      ```

      然后退出插入模式，输入:wq保存并退出

   3. 使命令文件生效

      ```c
      source ~/.bashrc
      ```

   vim中同理，想自定义宏、更改vim界面主题颜色的可以修改配置文件以达到效果，具体不会问豆包或者找我。

2. **gdb调试**

   1.编译工具链

   ```bash
   预处理阶段:gcc -E hello.c  -o hello.i
   编译阶段：gcc -S hello.i/hello.c -o hello.s
   汇编阶段：gcc -c hello.s -o hello.o
   链接 ：将多个.o文件，链接为一个可执行文件
   直接一步到位:gcc  hello.c -o hello 
   gcc 指令的其他常用选项
   -Wall 添加警告信息
   -O0,-O1,-O2,-O3编译器优化级别
   -O0表示不优化，-O1为默认值，开发时常选择，-O2为生产环境下常用的优化级别，-O3的优化级别最高。
   -g添加调试信息 (必加选项)
   ```

   2.调试程序

   ```c
   1.生成带调试信息的可执行程序指令
   2.进入GDB：
   	方式1-> gdb 文件名  
   	方式2-> gdb
   			file test1
   3.正式操作
   	3.1 list 查看代码 一次列出10行  l文件名:行号 在多个.c生成的可执行程序中，可以使用这个固定看哪个文件的多少行
   	3.2 run/r 启动程序
   	3.3 kill/k 终止调试，但是不会退出GDB
   	3.4 quit/q 退出gdb,调试也退
   4.断点
   	4.1 break/b 行号   b 文件名:行号   tb 临时断点
   	4.2 查看断点  info break / i b
   	4.3 所展示的信息
   		4.3.1 Num 唯一值，会不断自增，即使你删除一个断点，在建一个也不会从原来开始，而是会自增
   		4.3.2 disp --keep表示持久断点  --del表示一次性断点（程序运行过一次性断点后，断点就会删除）
   		4.3.3 enb 断点是否生效
   		4.3.4 what 断点是在哪个位置
   	4.4 删除断点 --d NUM 删除名为NUM 的断点   d 全部删除
   	4.5 运行
   		4.5.1 step/s 遇到函数会进去，不管你什么函数（不建议用这个）
   		4.5.2 next/n 遇到函数不会进去
   		4.5.3 c 跳到下一次断点
   		4.5.4 finsh/f 跳出函数  在main函数当中使用该指令是无效的
   		4.5.5 print/p express  打印某个表达式的值  也可以用其来修改变量
   		4.5.6 display/disp 将一个表达式得值持续挂到输出上 --- > undisplay/undisp  -- > info disp 查看有哪些持续得输出
   		4.5.7 info args ：输出参数  info locals 输出局部变量 
   		4.5.8 ignore N count 跳过N号断点count次，在循环的时候有用
   5.bt  显示函数调用栈
   6.传参数  ./test 11 22 33 
     		 r  参数1 参数2  参数3
   7.x查看内存单元的值
   	基本格式：x/(内存单元的个数)(内存数据的输出格式)(一个内存单元的大小) 数组名/指针/地址值...
   	该指令会从数组名/指针/地址值..开始，向后以(内存数据的输出格式)，展示(内存单元的个数) * (一个内存单元的大小)的内存数据。
       内存数据的输出格式有：
       o(octal)，八进制整数
       x(hex)，十六进制整数
       d(decimal)，十进制整数
       u(unsigned decimal)，无符号整数
       t(binary)，二进制整数
       f(float)，浮点数
       c(char)，字符
       a(address)，地址值
       s(string)，字符串
       一个内存单元的大小的表示，有以下格式：
       b(byte)，一个字节
       h(halfword,  2 bytes)，二个字节
       w(word, 4 bytes)，四个字节
       g(giant, 8 bytes)，八个字节
   ```
