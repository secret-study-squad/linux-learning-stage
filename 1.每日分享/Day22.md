1. **==缓冲区==** -- gzh

   1. **缓冲区的三种类型**

      | 类型   | 触发输出条件        | 典型对象 |
      | ------ | ------------------- | -------- |
      | 全缓冲 | **缓冲满 / fflush** | 文件     |
      | 行缓冲 | **遇到 `\n`**       | 终端     |
      | 无缓冲 | 立即输出            | stderr   |

   2. **stdout 的“动态变化”**

      第一次使用stdout时的上下文（它指向的是终端还是文件）将决定stdout的缓冲区刷新机制，并且在整个进程生命周期，这个刷新机制将默认保持不变：

      - 如果第一次使用stdout时它**指向一个终端**（比如用printf），此时缓冲区就是行缓冲的。
      - 如果第一次使用时stdout**被重定向到文件或其他非终端设备**，缓冲区就变成了满缓冲区，这意味着缓冲区会积累输出直到满了或者手动刷新。

   3. **fflush**——把“用户态缓冲区”的数据，强制写入到内核（文件描述符）

      ```c
      // printf → stdout缓冲区（用户态） → write(fd) → 内核缓冲（page cache） → 磁盘/终端
      ```

   4. **行缓冲**如果一直不遇到 `\n`，确实可能一直不刷新，**但是在以下情况仍会刷新：**

      - **缓冲区满**
      - 手动 fflush
      - **程序结束**
      - **发生输入操作时**（在进行输入操作前，C 标准库会自动刷新行缓冲的输出流，这是为了保证提示信息能够在程序阻塞等待输入之前显示出来，从而保证良好的交互体验，“程序要等你输入之前，必须先把话说完”）

   5. **补充：程序“正常结束”时会自动刷新(任何种类)缓冲区，但“异常结束”时不一定**

2. **mmap的相关注意事项** -- cjs

   ```c
   1.关于mmap函数其能够实现磁盘的某块区域与内存的某块区域进行映射，其修改内存就等价与修改磁盘文件，同时其可以实现随机访问
   2.参数列表：
   	addr 填NULL，其意思是自动找一片内存区域存放映射区（在堆上）
   	length 映射区的长度
   	prot mmap之前需要先调用open，open选项是O_RDWR，prot填PROT_READ|PROT_WRITE
   	flags 填 MAP_SHARED ，其意思是说如果再内存对文件进行了修改即会同时保存进入内存
   	fd 之前open得到的文件描述符
   	offset 即偏移量，从哪里开始偏移
   3.注意访问越界，当实现内存映射进行随机访问时，其最可能出现访问越界得情况，就会出现总线错误和段错误
   4.固定文件大小实现占位，当一个文件为空时，我们最好对其实行ftruncate确定好文件大小。
   ```

   - **`memcpy`**：
     - **功能**：用于拷贝**任意内存块**（不限于字符串）。它按字节进行拷贝，指定要拷贝的字节数，不关心内容，遇到`\0`也不会停止。
     - **安全性**：需要程序员自行保证拷贝的字节数不超过目标缓冲区大小。需要注意**内存重叠**问题，重叠时使用`memmove`更安全。
