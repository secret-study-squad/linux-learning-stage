1. 切换到 master 分支
git checkout master

2. 拉取远程 master 的最新代码（如果多人协作，这步必做）
git pull origin master

3. 合并 temp 分支到 master
git merge temp

4. 如果提示合并冲突，手动修改（保留正确代码，删除冲突标记），解决后，标记文件为已解决：
git add 冲突文件名  # 比如 git add app.py

5. 完成合并提交：
git commit -m "解决合并冲突，同步temp分支到master"

6. 推送合并后的 master 到远程
git push origin master
