# Linux学习阶段

## 0.最重要的注意事项

1. 首先，所有人只在**temp分支**进行拉取**pull**与上传**push**操作，如果修改**master分支**一定要在微信群里说一声。
2. 其次，每次要在temp分支操作前一定要有拉取pull操作，为了与仓库最新版本一致，并且仓库更新不pull一下也无法push。

## 1.每日分享

1. **分享任务：**

   **每人==每天==**最少分享一条有用的内容，最多3条。

2. **分享内容**

   何为有用——个人认为非常重要的即类似于高中时的**疑/难**点，并且觉得对大多数人都有帮助的内容

   **例：**Day17刚开始Linux阶段，但是所有人都不怎么熟悉Linux的快捷键操作，这样的话效率就会很低，所以如下所示分享了一组Linux快捷键操作

   | `Ctrl+A`     | 跳转到命令行开头         |
   | ------------ | ------------------------ |
   | **`Ctrl+E`** | **跳转到命令行结尾**     |
   | **`Ctrl+U`** | **清空光标到行首的内容** |
   | **`Ctrl+K`** | **清空光标到行尾的内容** |

3. **分享形式**

   对每天新建的md文件编辑，例如3.13的**Day17.md**文件，注意：所有人每天都对同一个md文件进行接力式操作。

4. **分享格式**

   上传时要备注一下是哪一天和哪个人，例如：

   ```c
   git add .
   git commit -m "Day17_gzh" // 天数+名字拼音缩写
   git push
   ```
